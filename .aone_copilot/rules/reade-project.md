# reade-project

## 项目文档

在开始任何任务之前，必须先读取项目根目录下的 `PROJECT_CONTEXT.md` 文件，该文件包含了整个项目的完整上下文（技术栈、目录结构、工具列表、数据流等），禁止通过扫描整个项目来了解项目结构。

## 文档同步

每次完成代码改动后，必须检查 `PROJECT_CONTEXT.md` 是否需要同步更新。以下情况必须更新文档：
- 新增或删除工具组件（需同步"工具列表"章节）
- 新增或修改公共组件（需同步"公共组件"章节）
- 修改目录结构（需同步"目录结构"章节）
- 修改核心配置文件（需同步"核心配置文件"章节）
- 新增或修改工具函数、常量、存储键（需同步对应章节）
- 修改数据流逻辑（需同步"关键数据流"章节）
